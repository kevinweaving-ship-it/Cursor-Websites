# Master Standings Process - All Classes

**⚠️ IMPORTANT: This process applies to ALL classes (Optimist A, Optimist B, ILCA 4, ILCA 6, Dabchick, etc.), not just Optimist B.**

The examples in this document use Optimist B and Optimist A, but the same rules, verification process, and methodology apply to every class in every regatta.

## CRITICAL RULES - MUST FOLLOW EXACTLY

### Rule 1: Main Regatta is the BENCHMARK/ANCHOR/YARDSTICK
- Main Regatta 339 (W. Cape Championships - Dinghy Classes, 26 Apr 2025) is the BENCHMARK
- All positioning is relative to main regatta sailors
- Main regatta sailors form the foundation of the standings

### Rule 2: Sailors NOT in Main Regatta
- **MUST be positioned BELOW all main regatta sailors**
- **UNLESS** they beat a main regatta sailor in a MORE RECENT regatta
- If they lost to all main regatta sailors they raced against, they go BELOW the last main regatta sailor

### Rule 3: Most Recent Regatta Trumps
- When comparing sailors, the most recent regatta result takes precedence
- If Sailor A beat Sailor B in Regatta 359 (Sep 2025) but lost to Sailor B in Regatta 306 (Feb 2025), the Sep 2025 result wins

### Rule 4: Head-to-Head is Required
- Sailors can only be positioned relative to each other if they raced together in the same regatta
- No assumptions or indirect comparisons

### Rule 5: Regatta Rankings Must Be 1st to X (Number of Entries)
- **CRITICAL:** For each class in any regatta, ranks MUST be **1st to X** where X = total number of entries for that class
- **NO text ranks** - only numeric ranks: 1, 2, 3, ... X
- **NO "UN-SEEDED"** as rank - assign actual numeric rank
- **NO "ZIM"** as rank - assign actual numeric rank
- **NO gaps or duplicates** - all entries must have consecutive numeric ranks 1, 2, 3, ... X
- **Example:** Optimist B in Regatta 375 has 18 entries → ranks must be 1-18 ONLY
- **Example:** Optimist A in Regatta 375 has 33 entries → ranks must be 1-33 ONLY

## STEP 9: MANDATORY CROSS-CHECK AND VERIFICATION

**⚠️ CRITICAL: THIS STEP IS MANDATORY FOR ALL CLASSES - DO NOT SKIP OR UPDATE DATABASE UNTIL THIS IS COMPLETE**

**This verification process applies to EVERY class in EVERY regatta:**
- Optimist A
- Optimist B  
- ILCA 4
- ILCA 6
- Dabchick
- RS Tera
- Topper
- Mirror
- **ALL OTHER CLASSES**

**You MUST verify rankings for EVERY class before updating the database. Do not assume any class is "simple" or skip verification.**

### Purpose:
- Ensure every sailor's rank is correct based on head-to-head results
- Prevent ranking errors that require later fixes
- Verify master standings logic has been applied correctly
- Catch issues BEFORE database update

### Process (MUST be done for EVERY class):

#### 1. For Each Ranked Sailor, Check Against ALL Sailors Ranked Above:

For each sailor at rank N, verify that:
- Every sailor ranked above them (ranks 1 to N-1) has either:
  - **Directly beaten them** in a head-to-head regatta, OR
  - **Beaten someone who beat them** (indirect comparison), OR
  - **Is in main regatta** and sailor is not (main regatta sailors anchor the list)

**SQL Query Template:**
```sql
-- For each sailor at rank N, check head-to-head with sailors ranked above
SELECT 
    reg.end_date,
    r1.rank as sailor_rank,
    r2.rank as other_rank,
    CASE 
        WHEN r1.rank < r2.rank THEN 'Sailor BEAT Other'
        WHEN r1.rank > r2.rank THEN 'Other BEAT Sailor'
        ELSE 'TIE'
    END as result
FROM results r1
JOIN results r2 ON r1.regatta_id = r2.regatta_id
JOIN regattas reg ON r1.regatta_id = reg.regatta_id
WHERE COALESCE(r1.helm_sa_sailing_id::text, 'TMP:' || r1.helm_temp_id::text) = '[sailor_id]'
AND COALESCE(r2.helm_sa_sailing_id::text, 'TMP:' || r2.helm_temp_id::text) = '[other_sailor_id]'
AND LOWER(COALESCE(r1.class_canonical, r1.class_original)) = '[class_name]'
AND LOWER(COALESCE(r2.class_canonical, r2.class_original)) = '[class_name]'
AND r1.raced = TRUE AND r2.raced = TRUE
ORDER BY reg.end_date DESC
LIMIT 1
```

#### 2. Specific Checks for Each Sailor:

**For each sailor ranked N:**
1. Get all sailors ranked above (1 to N-1)
2. For each sailor above, check:
   - **Most recent head-to-head result**
   - If sailor above has NEVER beaten sailor below → **PROBLEM**
   - If most recent result shows sailor below beat sailor above → **PROBLEM**
   - If no head-to-head exists → check indirect (beat someone who beat them)

#### 3. Common Issues to Catch:

**Issue 1: Sailor ranked too high**
- Example: Mika Malan ranked 4th, but has never beaten Sean (rank 5), Nathan (rank 7), Bazolele (rank 6), or Harold (rank 11)
- **Fix:** Move sailor below all sailors they've never beaten

**Issue 2: Most recent regatta trumps older result**
- Example: Ben Madel beat Mika Malan in most recent regatta (339), but Mika is ranked above Ben
- **Fix:** Move sailor with most recent win above the other

**Issue 3: Main regatta sailor below non-main regatta sailor**
- Example: Main regatta sailor (rank 3 in main) ranked below non-main regatta sailor
- **Fix:** Main regatta sailors must be above non-main regatta sailors (unless non-main beat them in newer regatta)

**Issue 4: Sailor who lost to all main regatta sailors ranked above main regatta sailor**
- Example: Sailor not in main regatta, lost to all main regatta sailors, but ranked above a main regatta sailor
- **Fix:** Move below all main regatta sailors

#### 4. Verification Checklist:

Before updating database, verify:

- [ ] **Every sailor ranked N has been checked against all sailors ranked 1 to N-1**
- [ ] **No sailor is ranked above someone who has always beaten them**
- [ ] **Most recent regatta results take precedence (newer trumps older)**
- [ ] **Main regatta sailors are positioned correctly (anchor positions)**
- [ ] **Sailors who lost to all main regatta sailors are below all main regatta sailors**
- [ ] **All head-to-head comparisons are from actual regatta results (same regatta, both raced)**
- [ ] **No assumptions made - if no head-to-head, position is justified (below main regatta or un-seeded)**
- [ ] **All ranks are numeric 1-X (no text, no gaps, no duplicates)**
- [ ] **Total entries = X, ranks are 1-X exactly**
